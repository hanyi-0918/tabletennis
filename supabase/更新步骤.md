# H2H 数据更新步骤（推荐流程）

**原则：后端 `backend/main.py` 是唯一数据源，Supabase 只做同步。**

---

## 一、改数据（只动一处）

1. **只改** `backend/main.py` 里的 `H2H_RECORDS`：
   - 加新比赛：在对应那对的 `recent` 列表**最前面**插入一条（格式与现有一致）
   - 改比分/胜场：改该条的文案，并改 `player1_wins` / `player2_wins`
   - 三大赛：如有需要再改 `player1_major_wins` / `player2_major_wins`

2. **不要**在 Supabase 里手写零散的 UPDATE，容易漏改或写错，导致数据乱。

---

## 二、同步到 Supabase（二选一）

### 方式 A：全量覆盖（最省心，推荐）

数据乱过、或改了好几对、或想保证和 backend 完全一致时用。

1. 在项目根目录执行：
   ```bash
   cd backend && python3 scripts/export_h2h_to_sql.py > h2h_seed.sql
   ```
2. 把新生成的 `backend/h2h_seed.sql` 覆盖到 supabase：
   ```bash
   cp backend/h2h_seed.sql supabase/h2h_reset_full.sql
   ```
3. 打开 **Supabase → SQL Editor → New query**，粘贴 **`supabase/h2h_reset_full.sql`** 的**全部内容**，点 **Run**。

结果：Supabase 里所有交手数据与 backend 一致。

### 方式 B：只更新某一对（可选）

只改了「樊振东 vs 马龙」且不想动其他行时：

1. 同样先执行 `export_h2h_to_sql.py` 生成 `h2h_seed.sql`。
2. 从 `h2h_seed.sql` 里找到**第一条** INSERT（樊振东, 马龙），复制出 `recent_matches` 和胜场那几列。
3. 在 Supabase SQL Editor 里执行一条 **UPDATE**（可参考 `supabase/update_fan_ma_h2h.sql` 的写法），只更新 `player1='樊振东' AND player2='马龙'` 的那一行。

不熟悉 SQL 时，**建议一直用方式 A**，不会乱。

---

## 三、前端赛事英文名（如需）

若在 `recent` 里加了**新赛事名称**（中文），在 **`frontend/src/App.jsx`** 的 `EVENT_ZH_TO_EN` 里加一条 `['中文名', 'English Name']`，这样英文模式下显示正确。

---

## 四、文件用途速查

| 文件 | 用途 |
|------|------|
| `backend/main.py` | **唯一数据源**，改交手只改这里 |
| `backend/scripts/export_h2h_to_sql.py` | 从 main.py 生成 SQL |
| `backend/h2h_seed.sql` | 脚本输出，全量 H2H 的 INSERT |
| `supabase/h2h_reset_full.sql` | 复制自 h2h_seed.sql，在 Supabase 里执行 = 全量覆盖 |
| `supabase/update_fan_ma_h2h.sql` | 只更新樊振东 vs 马龙一行的示例（可随 export 结果更新） |
| `supabase/schema.sql` | 建表、RLS、球员名单，**不要**往里塞 UPDATE |

---

**总结：以后做更新 = 改 main.py → 跑 export → 在 Supabase 执行 h2h_reset_full.sql（或单条 UPDATE）。**
